# Отчёт по практике

## Тема

Разработка системы обработки обращений клиентов (HelpDesk System).

## Цель работы

Целью работы является проектирование и разработка распределённого веб-приложения для управления клиентскими обращениями с применением FastAPI, React, PostgreSQL, Redis, Apache Kafka и Docker.

## Выполненные этапы

1. Выполнен анализ предметной области HelpDesk-систем.
2. Составлено техническое задание.
3. Спроектирована архитектура системы.
4. Разработана ER-диаграмма и структура базы данных PostgreSQL.
5. Подготовлена инфраструктура Docker Compose.
6. Реализован REST API на FastAPI.
7. Реализованы регистрация, вход, JWT-аутентификация и ролевая авторизация.
8. Реализована бизнес-логика жизненного цикла обращений.
9. Добавлено асинхронное взаимодействие через Apache Kafka.
10. Реализован сервис персональных уведомлений с хранением в PostgreSQL.
11. Реализован сервис аудита и административный журнал действий.
12. Добавлено Redis-кэширование статистики обращений.
13. Разработан веб-интерфейс на React с перепиской, уведомлениями и журналом аудита.
14. Добавлена административная панель обзора со статистикой, динамикой обращений и мониторингом PostgreSQL, Redis, Kafka и REST API.
15. Подготовлены тесты Pytest.
16. Подготовлены Dockerfile, docker-compose.yml и пример Kubernetes-манифестов.

## Демонстрация результатов

Для демонстрации проекта подготовлен сценарий `docs/demo_scenario.md`. Он включает проверку ролей, создание обращения пользователем, обработку обращения администратором, переписку, уведомления, журнал аудита, Kafka UI и RedisInsight.

Ключевые демонстрируемые сценарии:

- пользователь видит только свои обращения;
- исполнитель видит только назначенные ему обращения;
- суперадминистратор видит все обращения, управляет ролями и журналом аудита;
- исполнитель выбирается только из активных пользователей с ролью `AGENT`;
- переходы статусов ограничены жизненным циклом `OPEN → IN_PROGRESS → RESOLVED → CLOSED`;
- уведомления создаются асинхронно через Kafka;
- статистика обращений кэшируется в Redis.

- FastAPI;

- PostgreSQL;
- Redis;
- Apache Kafka;
- React;
- Docker и Docker Compose;

## Результат

В результате разработан проект, включающий backend, frontend, сервис уведомлений, сервис аудита, инфраструктурные конфигурации и документацию. Приложение можно запустить локально командой `docker compose up --build`, после чего API доступен по адресу `http://localhost:8000/docs`, а веб-интерфейс — по адресу `http://localhost:3000`.
